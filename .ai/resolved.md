## R-003: AI-провайдери для текстового збагачення
**Дата:** 2026-04-13
**Питання:** Який AI-сервіс для збагачення карток?
**Рішення:** **Мульти-провайдер: Anthropic API + OpenAI API + Google AI (Gemini)**. Три варіанти для flexibility і fallback.
**Чому:** Провайдери оновлюються, якість змінюється — потрібна гнучкість
**Хто прийняв:** user

## R-004: Формат XML-експорту для партнерів
**Дата:** 2026-04-13
**Питання:** Як партнери отримують дані?
**Рішення:** **2 статичних XML файли (товари + категорії) по публічному URL, без авторизації, тільки товари in_stock, без поля quantity**
**Чому:** Простий механізм, партнерам не потрібен логін — просто URL
**Хто прийняв:** user

## R-005: Ролі для v1
**Дата:** 2026-04-13
**Питання:** Які ролі потрібні в першій версії?
**Рішення:** **admin** (повний доступ) + **operator** (редагування карток, категорій, ревю AI). Майбутні: manager (view-only), гнучкі permissions.
**Чому:** Мінімум достатній — один адмін + оператор(и) на потоці
**Хто прийняв:** user

## R-013: Механізм доставки XML з BUF
**Дата:** 2026-04-13
**Питання:** Як XML-файли потрапляють з BUF в DataPIM?
**Рішення:** **З папки (shared folder).** BUF автоматично кладе файли в папку, DataPIM їх звідти забирає.
**Чому:** Найпростіший механізм, не потребує API чи FTP
**Хто прийняв:** user

## R-014: Стратегія імпорту товарів
**Дата:** 2026-04-13
**Питання:** Імпортувати всі 36К товарів чи тільки in_stock?
**Рішення:** **BUF вигружає все як є. Ми:** (1) створюємо нові картки тільки для in_stock=true, (2) існуючі в БД товари — оновлюємо флаг in_stock, (3) ніколи не видаляємо товари з БД.
**Чому:** Не засмічуємо БД 24К неактуальних позицій. Збагачення AI/оператора не втрачається коли товар тимчасово зникає.
**Хто прийняв:** user

## R-015: Дедуплікація категорій
**Дата:** 2026-04-13
**Питання:** Як обробляти 1443 дублікати в XML категорій?
**Рішення:** **Upsert по unique ID** — при імпорті дублікати просто ігноруються (перезаписуються)
**Чому:** XML має ~50% дублікатів, upsert — найпростіше рішення
**Хто прийняв:** user/agent

## R-016: Сервіси генерації зображень
**Дата:** 2026-04-13
**Питання:** Які AI-сервіси використовувати для генерації продуктових зображень?
**Розглянуті варіанти:**
- Flux Pro (BFL/Replicate) — фотореалізм на рівні Midjourney, REST API, $0.04–0.06
- DALL-E 3 (OpenAI) — стабільна якість, вже інтегруємо OpenAI для тексту, $0.04–0.12
- Imagen 3 (Google Vertex AI) — відмінний фотореалізм, потребує GCP, $0.03–0.06
- Midjourney v6 — найкраща якість, але **немає API** — відкинутий
- Stable Diffusion, Leonardo, Ideogram — нижча якість для продуктових фото
**Рішення:** **Flux Pro (основний) + DALL-E 3 (fallback) + Imagen 3 (альтернатива)**
**Чому:** Три провайдери з REST API, різні екосистеми для надійності, оптимальна якість для промислових товарів
**Хто прийняв:** user
