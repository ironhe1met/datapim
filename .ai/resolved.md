## R-014: Стратегія імпорту товарів
**Дата:** 2026-04-13
**Питання:** Імпортувати всі 36К товарів чи тільки in_stock?
**Рішення:** **BUF вигружає все як є. Ми:** (1) створюємо нові картки тільки для in_stock=true, (2) існуючі в БД товари — оновлюємо флаг in_stock, (3) ніколи не видаляємо товари з БД.
**Чому:** Не засмічуємо БД 24К неактуальних позицій. Збагачення AI/оператора не втрачається коли товар тимчасово зникає.
**Хто прийняв:** user

## R-015: Дедуплікація категорій
**Дата:** 2026-04-13
**Питання:** Як обробляти 1443 дублікати в XML категорій?
**Рішення:** **Upsert по unique ID** — при імпорті дублікати просто ігноруються (перезаписуються)
**Чому:** XML має ~50% дублікатів, upsert — найпростіше рішення
**Хто прийняв:** user/agent

## R-016: Сервіси генерації зображень
**Дата:** 2026-04-13
**Питання:** Які AI-сервіси використовувати для генерації продуктових зображень?
**Розглянуті варіанти:**

**Рішення:** **Flux Pro (основний) + DALL-E 3 (fallback) + Imagen 3 (альтернатива)**
**Чому:** Три провайдери з REST API, різні екосистеми для надійності, оптимальна якість для промислових товарів

## R-018: AI як опціональний модуль
**Дата:** 2026-04-14
**Питання:** Як архітектурно інтегрувати AI фічі, якщо вони не входять в v1.0?
**Рішення:** **AI — це опціональний модуль.** В Settings буде перемикач "Увімкнути AI модуль". Коли ON: з'являється таб "AI" на картці товару, "Ревю AI" в sidebar, кнопки AI-enrichment. Коли OFF: всі AI елементи приховані.
**Чому:** PIM може працювати без AI (ручне збагачення). AI — додаткова зручність. Модульний підхід дозволяє клієнтам вибирати.
**Версія:** Підготовка архітектури — v1.0. Повна реалізація AI — v1.2+.
**Хто прийняв:** user

## R-017: Override pattern для полів товару
**Дата:** 2026-04-13
**Питання:** Як захистити збагачені дані від перезапису при імпорті з BUF?
**Розглянуті варіанти:**
- Прапорець is_edited на кожне поле — працює, але крихкий
- Override pattern (buf_* + custom_*) — стандарт PIM індустрії
**Рішення:** **Override pattern.** Поля з BUF мають два варіанти: `buf_name` (завжди оновлюється з BUF) + `custom_name` (nullable, редагується оператором/AI). Відображення: `custom ?? buf`. Імпорт завжди пише в buf_*, ніколи не чіпає custom_*. Поля тільки DataPIM (description, seo, характеристики) — без buf_* префікса.
**Чому:** Надійніший за прапорці, простий імпорт, завжди є оригінал для порівняння, можна "скинути до оригіналу"
**Хто прийняв:** user
