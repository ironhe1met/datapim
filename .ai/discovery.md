# Discovery Brief

## 1. Elevator Pitch
> **DataPIM** — це PIM-система для Meta Group, яка перетворює голі товарні дані з BUF в повноцінні продуктові картки завдяки AI-збагаченню, і автоматично віддає готовий каталог партнерам через XML.

## 2. Суть проєкту
- **Назва:** DataPIM
- **Компанія:** Meta Group
- **Тип:** продукт (internal tool → B2B в майбутньому)
- **Проблема:** Товарні картки з BUF приходять "голими" — тільки назва, код, ціна. Немає інструменту для збагачення карток контентом і дистрибуції партнерам. Зараз нульовий процес — ніхто цим не займається.
- **Валідація проблеми:** підтверджена — власний досвід, реальна потреба бізнесу
- **Рішення:** Веб-сервіс який імпортує XML з BUF, дає змогу оператору та AI-агенту збагачувати картки (описи, характеристики, зображення), і експортує готовий каталог партнерам через публічний XML

## 3. Цільова аудиторія
- **Основна:** Оператор(и) компанії Meta Group — нетехнічні користувачі, потоковий ввід/редагування карток
- **Вторинна:** Адмін (власник) — технічний, налаштування, автоматизація, управління
- **Кількість користувачів:** 1–5 (внутрішній інструмент)
- **Технічний рівень:** Оператор — нетехнічний (простий інтерфейс). Адмін — технічний.

## 4. Ролі та доступ
| Роль | Що може | Реєстрація |
|------|---------|------------|
| admin | Повний доступ: користувачі, налаштування, імпорт/експорт, редагування, AI, ревю | seed (початковий) |
| operator | Редагування карток, категорій, ревю AI-змін, перегляд каталогу | invite від admin |

Майбутні ролі (не в v1):
- `manager` — тільки перегляд каталогу
- Гнучка система прав (налаштовувані permissions)

- **Multi-tenancy:** Ні. Одна компанія, один каталог. B2B multi-tenancy — в v2.
- **Управління користувачами:** Admin може створювати, редагувати логін/пароль/email/ім'я

## 5. Ключові функції (MVP / v1.0.0)
- [ ] XML-імпорт товарів та категорій з BUF (автоматично, 1+ разів/день)
- [ ] Імпорт картинок (файли, прив'язка до товарів по коду)
- [ ] Повна авторизація (JWT) + RBAC (admin, operator)
- [ ] Управління користувачами (CRUD, invite, зміна паролю)
- [ ] Каталог товарів з фільтрами, пошуком, пагінацією
- [ ] Управління категоріями (ієрархія, перейменування, редагування)
- [ ] Картка товару — повне редагування (назва, опис, характеристики, бренд, картинки)
- [ ] AI-агент для автозбагачення карток (опис, характеристики, SEO) — мульти-провайдер
- [ ] AI-генерація зображень для товарів — мульти-сервіс
- [ ] Модуль ревю: AI редагує → оператор підтверджує → картка в production
- [ ] XML-експорт для партнерів: 2 файли (товари + категорії), тільки in_stock, публічний URL
- [ ] Dark/Light mode
- [ ] Українська мова UI (i18n-архітектура ready для EN)
- **Одна головна:** Імпорт + AI-збагачення карток + XML-експорт (повний цикл даних)

## 6. Що НЕ входить в MVP
- B2B модуль (вхід партнерів, замовлення, кабінет)
- Multi-tenancy / SaaS
- Англійський інтерфейс (але i18n закладено)
- Роль manager (тільки перегляд)
- Гнучка система permissions
- Мобільний додаток
- Кошик, оплата, доставка
- Аналітика / статистика продажів
- Мультимовні картки товарів

## 7. Дані та сутності
| Сутність | Приблизний обсяг | Ріст | Чутливість |
|----------|-----------------|------|------------|
| Products | ~36 000 (all), ~10 000 (in stock) | Повільний, +десятки/тиждень | — |
| Categories | ~2 920 (з дублікатами в XML) | Рідко змінюються | — |
| Images | 21 (тест), потенційно тисячі | Зростає з збагаченням | — |
| Users | 1–5 | Фіксований | PII (email, password) |
| AI Tasks | Десятки/день | Зростає | — |
| AI Reviews | Десятки/день | Зростає | — |

- **Міграція:** Початковий імпорт з XML (TMC.xml — товари, TMCC.xml — категорії)
- **Пошук/фільтрація:** Потрібен — по назві, категорії, in_stock, бренду, ціні
- **Відомі проблеми XML:**
  - Дублікати категорій (один id кілька разів)
  - Пусті поля (brand, country_of_origin, quantity)
  - Формат ціни "1 015,16" (пробіл як розділювач тисяч, кома для копійок)
  - SKU з trailing spaces
  - BOM у файлі (UTF-8 BOM)

## 14. Дедлайни та ризики
- **Хард-дедлайн:** 2026-04-14 (завтра) — перша робоча версія
- **Припущення:**
  - XML-формат з BUF стабільний і не зміниться без попередження
  - AI API доступні та мають достатні ліміти
  - Один оператор на старті
- **Ризики:**
  - Якість XML: дублікати категорій, пусті поля, формат цін — потрібен robust парсер
  - Зміна формату XML з BUF без попередження
  - AI API costs при масовому збагаченні 36К карток
  - Залежність від зовнішніх AI API (downtime, rate limits)
- **Регуляторні вимоги:** Немає (крім базового захисту PII користувачів)

## 15. Метрики успіху
- Веб-інтерфейс показує всі товари з BUF з можливістю редагування
- AI-агент збагачує картки (опис, характеристики, зображення)
- Оператор може ревюїти AI-зміни
- XML-експорт працює — партнери отримують 2 файли по публічному URL
- Імпорт/експорт працює автоматично

## Handoff → наступний агент
- **Pipeline:** app
- **Складність:** Середня-висока
  - 2 ролі (admin, operator) + гнучкі permissions в майбутньому
  - 6+ сутностей (products, categories, images, users, ai_tasks, ai_reviews)
  - 7+ інтеграцій (BUF XML, 3 AI text APIs, 3-4 AI image APIs)
  - File upload (images)
  - Background jobs (AI enrichment, XML import/export)
  - PII дані (users) → потрібен Security агент
- **Наступний агент:** `prd`
- **Головний фокус:** Потік даних: BUF XML → Import → DB → AI Enrichment → Review → Export XML
- **Ключові ролі:** admin, operator
- **Ключові сутності:** Product, Category, Image, User, AITask, AIReview
- **Ключові інтеграції:** BUF XML import, Anthropic/OpenAI/Google AI APIs, Image Generation APIs
- **Red flags:**
  - Дедлайн дуже стислий (1 день) — потрібна чітка пріоритизація
  - XML з BUF має проблеми якості (дублікати, формати)
  - Механізм доставки XML з BUF не визначений
- **Складні місця:**
  - Дедуплікація категорій при імпорті
  - Парсинг цін з форматом "1 015,16"
  - AI review workflow (draft → review → production)
  - Мульти-провайдер AI routing

- **Обмеження:** Docker + nginx + Ubuntu 22, Ukrainian UI, Material Design
